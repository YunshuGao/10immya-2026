Typography on the Web

Week 6 · 75 minutes · Year 10 Industrial Technology — Multimedia

IND5-2IND5-4IND5-5

Learning intentions

  • I can import a Google Font and use it in my CSS.
  • I can set a consistent type scale for headings and body.
  • I can make text readable with line-height and line-length.

Success criteria

  • I have imported one Google Font and used it in my CSS.
  • My headings (h1, h2, h3) have a clear size hierarchy.
  • My body text has line-height 1.5-1.7 and line-length 60-75 characters.

Do Now · 5 min

Read two news articles — one with comfortable typography, one that feels 'off'. What makes the readable one readable?

Hint: it's not just the font. It's also:

  • How big is the text?
  • How much space between lines?
  • How wide is the column? (Too narrow = choppy. Too wide = eye strain.)

I Do · ~10 min Teacher demonstration

Teacher demonstrates importing a Google Font:

<!-- in the head of HTML -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap" rel="stylesheet">
body {
  font-family: 'Inter', system-ui, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  max-width: 65ch;    /* 65 characters wide is optimal for reading */
  margin: 0 auto;
  padding: 2rem;
}

h1 { font-size: 2.5rem; font-weight: 700; line-height: 1.2; }
h2 { font-size: 1.75rem; font-weight: 600; }
h3 { font-size: 1.25rem; font-weight: 600; }
The industry rules of readable type:
  1. Body text: 16px minimum (larger is often better)
  2. Line-height: 1.5-1.7 for body (not too tight, not too loose)
  3. Line length: 60-75 characters per line (use max-width: 65ch)
  4. Only 2 fonts max (one heading, one body — or the same font at different weights)

We Do · ~15 min Guided build

Together, pair Inter (sans-serif) with Playfair Display (serif) for a classic editorial look:

<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600&family=Playfair+Display:wght@700&display=swap" rel="stylesheet">
body { font-family: 'Inter', sans-serif; }
h1, h2, h3 { font-family: 'Playfair Display', serif; }

Discuss: when does this pairing work? When does a single-font approach work better?

You Do · ~35 min Independent practice

Update your portfolio styles.css to use typography thoughtfully:

  1. Choose ONE font for body (sans-serif recommended for UI)
  2. Choose ONE font for headings (can be the same, or a contrasting serif)
  3. Set a type scale: h1 is biggest, h2 medium, h3 smaller, body is 16-18px
  4. Set line-height: 1.6 on body
  5. Set max-width: 65ch on a reading container
  6. Test on your phone — is it readable?

Submit a "type spec" screenshot: your h1, h2, h3, and body paragraph with font names labelled.

Differentiation

Support

3 pre-selected font pairings provided. Student picks one and applies.

Core

Choose fonts from Google Fonts independently.

Extension

Create a full type scale using a ratio (e.g. 1.25, 1.5). Research 'modular scale' on type-scale.com.
